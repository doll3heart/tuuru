# Tuuru 新窗口交付说明（2026-07-22）

## 给新窗口的任务

继续维护 `D:\Projects\Tuuru`。这是纯本地互动文章／小手机阅读器与编辑器，没有业务后端；作者和读者的数据主要保存在各自浏览器本地，作品通过 JSON／PNG 文件传播。

本窗口已经完成社交系统这一批改动。下一窗口请集中制作一个小型入口页面，分成：

1. **写作习惯**：联系人跨作品导入／导出、作者占位符习惯的全局设置。
2. **使用教程**：内置教程和功能名词说明，尤其解释别名、小号、固定脸／视频通话背景、占位符等容易误解的功能。

入口按钮要小巧，放在创作端／读者端切换按钮附近，不要挤占主操作区。先检查现有首页结构和移动端空间，再给出实现；不要重新发明一套与现有灰粉 UI 冲突的样式。

## 本窗口已完成

### 论坛 IP

- 增加论坛 IP 显示开关，默认关闭。
- 开启后，作者端和读者端的帖子、评论与楼中楼可显示角色配置的 IP。
- 联系人主号、小号和论坛 NPC 均可设置 IP。
- 读者本人回复不会被伪造一个 IP。

### 联系人排序

- 支持置顶。
- 支持 A–Z 排序和自定义排序。
- 自定义模式支持拖拽以及键盘上下移动。
- 文章内置小手机和独立小手机的联系人编辑入口都已接入。
- 读者端联系人列表遵循作者保存的顺序。

### 身份头像

- 消息头像与论坛头像已完全拆分。
- 联系人仍保留通用头像；旧作品缺少新字段时自动回退到通用头像，不破坏旧数据。
- 相关字段为 `messageAvatarUrl`、`forumAvatarUrl`，旧字段为 `avatarUrl`。
- 原“固定脸 URL”目前承担**视频通话背景图**用途；新窗口做教程时应明确写出这个用途，避免用户误解。

### 读者互动续答

- 读者选择一条回复后，作者可以配置多个后续回复。
- 每条后续回复可选择不同联系人、联系人小号或论坛 NPC。
- 旧作品中没有角色信息的后续消息仍有兼容回退。

### @ 提及

- 群聊与论坛的发帖、评论、回复和编辑流程均可选择角色并插入 `@名称`。
- 作者端预览、读者端论坛、内联论坛和群聊都会高亮识别到的提及。
- 存储和导出仍是纯文本，不把样式 HTML 写进作品文件。

### 同期已处理的相关反馈

- 论坛评论／回复支持作者调整顺序；相关逻辑位于 `js/forum-comment-reorder.js`。
- 浏览器历史记录的日期时间可由作者编辑，不再自动覆盖。
- 联系人通用头像、消息头像、论坛头像和视频通话背景的职责已经分开。
- 圆形头像读取与论坛／消息身份解析已有回归测试覆盖。

## 数据兼容约束

- 不要重命名或删除旧字段；新增字段必须有默认值或旧字段回退。
- 不要清空、迁移或覆盖浏览器 IndexedDB／localStorage 来“解决”显示问题。
- 源码更新本身不应删除作者或读者的本地作品；但浏览器清站点数据、隐私清理、换域名／端口会形成不同的本地存储空间。
- 作品导入、JSON 导出和 PNG 导出必须保持同一语义。涉及作品结构时至少运行 `tests/work-transport-parity.test.mjs`。
- 联系人跨作品导入／导出建议采用显式、带版本号的数据包；导入时合并并处理 ID 冲突，禁止整库覆盖。
- 作者全局习惯应该单独保存在本地设置命名空间，不应悄悄写入或重写每一篇现有作品。
